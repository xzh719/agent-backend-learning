1工具注册与发现

Agent 的工具不是写死的字符串数组。一个成熟的系统需要动态注册、发现、管理工具。

Tool Registry

from typing import Callable, Optional
from dataclasses import dataclass

@dataclass
class ToolDef:
    name: str
    description: str
    parameters: dict
    handler: Callable
    category: str = "general"
    require_auth: bool = False
    require_sandbox: bool = False

class ToolRegistry:
    """全局工具注册表 —— 单一数据源"""
    def __init__(self):
        self._tools: dict[str, ToolDef] = {}

    def register(self, tool: ToolDef):
        self._tools[tool.name] = tool

    def get_openai_format(self, category=None) -> list[dict]:
        tools = self._tools.values()
        if category:
            tools = [t for t in tools if t.category == category]
        return [{"type":"function", "function":{"name":t.name, "description":t.description, "parameters":t.parameters}} for t in tools]

    async def execute(self, name, args):
        tool = self._tools[name]
        if tool.require_sandbox:
            return await sandbox.run(tool.handler, args)
        return await tool.handler(**args)

2Docker 沙箱执行

让 LLM 直接在你服务器上执行代码 = 自爆。Docker 沙箱是 Agent 代码执行的安全底线。

沙箱设计原则

┌──────────────────────────────────┐ │ Host (你的 Agent 服务器) │ │ │ │ ┌──────────────────────────────┐ │ │ │ Docker Container (沙箱) │ │ │ │ - 网络: 无/仅内网 │ │ │ │ - 文件: tmpfs (内存,退出即删) │ │ │ │ - CPU: 限制 1 核 │ │ │ │ - 内存: 限制 256MB │ │ │ │ - 超时: 30s │ │ │ │ - 权限: no-new-privileges │ │ │ │ - 只读 rootfs │ │ │ └──────────────────────────────┘ │ └──────────────────────────────────┘

Python Docker SDK 沙箱

import docker

client = docker.from_env()

SANDBOX_IMAGE = "python:3.11-slim"
SECURITY_OPTS = [
    "no-new-privileges",
    "seccomp=unconfined",
]

async def execute_in_sandbox(code: str, timeout=30) -> str:
    container = client.containers.run(
        image=SANDBOX_IMAGE,
        command=["python", "-c", code],
        detach=True,
        network_disabled=True,      # 断网
        mem_limit="256m",              # 内存限制
        nano_cpus=1_000_000_000,      # 1 CPU 核
        read_only=True,              # 只读文件系统
        tmpfs={"/tmp": "size=64M"}, # 临时可写区
        security_opt=SECURITY_OPTS,
        remove=True,                  # 跑完自动删除
    )
    try:
        result = container.wait(timeout=timeout)
        logs = container.logs().decode()
        return logs
    except docker.errors.APIError:
        container.kill()
        raise
    finally:
        container.remove(force=True)
绝对不要在宿主机上执行 LLM 生成的代码。即使用了 Docker 沙箱，也要限制网络、CPU、内存、超时，并设只读文件系统。

3权限与安全模型

Agent 能做的事情越来越多，必须有精细的权限控制——谁能用什么工具、读什么文件、调什么 API。

RBAC + Scope 权限模型

class PermissionManager:
    """RBAC: 角色 → 权限 → 工具/操作"""
    ROLES = {
        "admin": {"tools:*", "files:*", "system:*"},
        "user":  {"tools:search", "tools:read_file", "files:read"},
        "guest": {"tools:search"},
    }
    def can(self, role, permission):
        allowed = self.ROLES.get(role, set())
        return permission in allowed or f"{permission.split(':')[0]}:*" in allowed

安全防线层级

层防线作用
L1工具白名单只暴露注册过的工具
L2参数校验Pydantic 校验参数类型/范围
L3权限检查用户角色决定可见工具
L4资源限制超时/内存/网络/文件数
L5沙箱隔离Docker/gVisor/Firecracker
L6审计日志所有操作可追溯

4外部 API 集成

Agent 需要操作真实世界的服务——GitHub、Slack、Jira、Gmail。每个 API 都有不同的认证和交互模式。

统一集成模式

Agent Tool Definition → Integration Adapter → External API │ ┌───────────────┼───────────────┐ ▼ ▼ ▼ OAuth 2.0 API Key Webhook (用户授权) (服务级) (事件驱动) 典型场景: - GitHub: OAuth → 用户给 Agent 授权 → Agent 代操作 - SendGrid: API Key → 服务级发邮件 - Stripe: Webhook → Agent 收到支付通知后自动处理

OAuth Token 管理

class OAuthManager:
    """管理用户的三方 OAuth Token"""
    async def get_valid_token(self, user_id, provider):
        token = await db.get_token(user_id, provider)
        if token.is_expired():
            new_token = await self.refresh(token.refresh_token, provider)
            await db.save_token(user_id, provider, new_token)
            return new_token.access_token
        return token.access_token

# GitHub 集成工具示例
async def github_create_issue(user_id, repo, title, body):
    token = await oauth.get_valid_token(user_id, "github")
    async with httpx.AsyncClient() as client:
        resp = await client.post(
            f"https://api.github.com/repos/{repo}/issues",
            headers={"Authorization": f"Bearer {token}"},
            json={"title": title, "body": body},
        )
        return resp.json()

5插件系统设计

让第三方开发者为你的 Agent 贡献工具。一个好的插件系统 = 生态护城河。

插件协议 (manifest.json)

{
  "name": "weather",
  "version": "1.0.0",
  "tools": [{
    "name": "get_weather",
    "description": "获取城市天气",
    "parameters": { "type": "object", "properties": { "city": {"type": "string"}}}
  }],
  "entry": "main.py",
  "permissions": ["network:api.weather.com"]
}
插件系统三个核心能力：① 动态加载/卸载 ② 隔离执行（每个插件独立进程或 Docker）③ 权限声明（manifest 中声明需要什么权限，用户授权后才启用）。

1Docker 容器化

Agent 后端依赖 Python 版本、系统库、GPU 驱动... 用 Docker 一键打包，到处运行。

Dockerfile 最佳实践

# 多阶段构建 —— 最终镜像尽量小
FROM python:3.11-slim AS builder
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

FROM python:3.11-slim
RUN useradd --create-home agentuser
USER agentuser
WORKDIR /app
COPY --from=builder /usr/local/lib/python3.11/site-packages /usr/local/lib/python3.11/site-packages
COPY . .
EXPOSE 8000
HEALTHCHECK --interval=30s --timeout=5s CMD curl -f http://localhost:8000/health || exit 1
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

Docker Compose 全家桶

2消息队列集成

Agent 的 LLM 调用可能耗时几十秒。不能让用户的 HTTP 请求干等。消息队列实现异步任务处理。

Celery + Redis 异步任务

from celery import Celery

celery_app = Celery('agent', broker='redis://redis:6379/0')

@celery_app.task(bind=True, max_retries=3)
def run_agent_task(self, session_id, user_input):
    try:
        agent = AgentLoop(...)
        return await agent.run(user_input)
    except Exception as e:
        self.retry(exc=e, countdown=10)

# FastAPI 路由: 立即返回 task_id，由 WebSocket 推送结果
@app.post("/tasks")
async def create_task(req: TaskRequest):
    task = run_agent_task.delay(req.session_id, req.input)
    return {"task_id": task.id}

4CI/CD 流水线

手动部署 = 人肉出错。CI/CD 让测试、构建、部署全自动化。

Push to main ──> CI Pipeline: ① Lint (ruff, mypy) ② Unit Tests (pytest) ③ Eval Tests (Agent accuracy check) ④ Docker Build ⑤ Push to Registry ⑥ Deploy (kubectl apply / docker compose up -d) ⑦ Smoke Test (health check + 基础功能验证) ⑧ Notify (Slack/Discord)

GitHub Actions 示例

name: Deploy Agent
on:
  push:
    branches: [main]
jobs:
  test-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: {python-version: "3.11"}
      - run: pip install -r requirements.txt && pytest
      - run: docker build -t agent-api .
      - run: docker push registry.example.com/agent-api:latest
